# Preprocessing Guide

This document describes the image/mask and radiology report preprocessing pipelines used to prepare the training data. The scripts are located in `RADAR_train/preprocess_code/`.

> **Prerequisite:** Complete the [Setup](../README.md#setup) section in the main README first.

---

## External Dependencies

Our preprocessing pipeline relies on the following external tools/services. **They are not bundled with this repository and must be obtained/configured separately.**

| Dependency                                                                     | Type                          | Used by                                                                                                | Notes                                                                                                                            |
| ------------------------------------------------------------------------------ | ----------------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------- |
| [TotalSegmentator V1](https://github.com/wasserth/TotalSegmentator/tree/v1.5.7) | Open-source segmentation tool | /                                                                                                      | Generates the raw 104-structure organ segmentation masks. Must be installed and run separately to produce masks before merging.  |
| DashScope / Qwen / Other LLM                                                   | API                           | Report preprocessing (`check_organ_mention.py`, `report_parsing.py`, `report_parsing_normal.py`) | Requires a valid API key. Set`dashscope.api_key` in each script before running. Alternatively, you may use other LLM and APIs. |

> **Note:** The pretrained masks we release already went through the TotalSegmentator step, so you only need TotalSegmentator if you want to preprocess your own images from scratch. Likewise, the report-parsing scripts are only needed if you want to reproduce our report processing on custom data; any other LLM can be substituted by adapting the API calls in these scripts.

## 1. Image and Mask Preprocessing

We use [TotalSegmentator V1](https://github.com/wasserth/TotalSegmentator/tree/v1.5.7) to generate segmentation masks for 104 anatomical structures. The preprocessing then maps these masks to 36 major anatomical structures and resamples both the images and masks to a spacing of `[1, 1, 5]`.

### Step 0: Generate raw masks with TotalSegmentator (external)

Install and run [TotalSegmentator V1](https://github.com/wasserth/TotalSegmentator/tree/v1.5.7) on the original CT volumes to produce the 104-structure segmentation masks. This step is external and not included in this repository.

### Step 1: Merge anatomies and resample (image + mask)

`process_img_mask.py` merges the 104 TotalSegmentator structures into 36 major anatomical structures, then resamples both the image and the merged mask to a spacing of `[1, 1, 5]`.

```bash
cd RADAR_train/preprocess_code
python process_img_mask.py \
    --src-dir /path/to/totalsegmentator_masks \
    --root-dir /path/to/merlin_data_root
```

- `--src-dir`: directory containing the raw masks generated by TotalSegmentator.
- `--root-dir`: directory containing the original MERLIN data. Outputs (`merged_masks/`, `resized_images/`, `resized_masks/`) are written under the root directory.
- `--slice-id` / `--num-slices` (optional): shard the workload across multiple processes/machines for parallel processing.

### (Alternative) Resample images only

If the processed masks are already available (e.g. our released masks) and you only need to resample the original images to `[1, 1, 5]`, use `process_img.py`:

```bash
cd RADAR_train/preprocess_code
python process_img.py \
    --src-dir /path/to/resized_masks \
    --root-dir /path/to/merlin_data_root
```

- `--src-dir`: directory of our processed masks (used to enumerate the cases to resize).
- `--root-dir`: directory containing the original MERLIN data; resized images are written to `resized_images/` under the root directory.

## 2. Report Preprocessing

Report preprocessing extracts anatomy-level supervision from free-text radiology reports. All three scripts call the **DashScope/Qwen API**. Before running, set your API key in each script:

```python
dashscope.api_key = "YOUR_DASHSCOPE_API_KEY"
```

The input is a report file `ckpt/merlin_report.json`. Run the scripts in the following order, as each step depends on the output of the previous one.

### Step 1: Organ mention detection

`check_organ_mention.py` determines whether a report mentions each abdominal anatomical structure.

```bash
cd RADAR_train/preprocess_code
python check_organ_mention.py
```

- Output: `merlin_report_mention.json`

### Step 2: Anatomy-level description extraction

`report_parsing.py` extracts the anatomy-level description for each mentioned structure.

```bash
cd RADAR_train/preprocess_code
python report_parsing.py
```

- Input: `merlin_report_mention.json`
- Output: `merlin_report_organ_report_v1.json`

### Step 3: Normal/abnormal classification

`report_parsing_normal.py` determines whether each mentioned structure is described as **normal** or **abnormal**. This information is used to reduce false negatives during training.

```bash
cd RADAR_train/preprocess_code
python report_parsing_normal.py
```

- Input: `merlin_report_mention.json`
- Output: `merlin_report_organ_normal_v1.json`
